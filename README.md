# 小型双臂复合机器人系统

这是一个面向工程实训场景的小型双臂复合机器人项目。系统以 ESP32-S3 为车载主控，连接地面站、Jetson Nano 视觉端、STM32F103 麦克纳姆轮底盘和众灵 ZP 总线舵机控制板，实现人工遥控、底盘运动转发、双机械臂动作组执行、视觉识别结果回传、心跳状态上报和急停保护。

项目重点不只是单个硬件驱动，而是把多控制器、多串口、多协议设备整合成一套可联调、可扩展的嵌入式机器人控制系统。

## 功能特性

- **地面站控制**：ESP32-S3 + TJC 串口屏读取按钮事件，通过无线串口向车载主控发送统一控制帧。
- **车载中心路由**：ESP32-S3 主控接收地面站和 Jetson 数据，按功能码分发到底盘、机械臂或状态管理模块。
- **麦克纳姆轮底盘控制链路**：底盘运动命令由车载 ESP32-S3 转发给 STM32F103，支持前进、后退、左右平移、旋转、停止和矢量移动。
- **双机械臂控制**：封装众灵 ZP 总线舵机 ASCII 协议，通过预设动作组执行复位、夹取、放下、停止、释力和恢复扭力。
- **视觉信息接入**：Jetson Nano 可通过统一协议上报物体类型和数量，主控更新状态并转发给地面站显示。
- **统一通信协议**：自定义二进制变长帧，包含设备地址、消息类型、功能码、序列号、数据区、CRC-16/MODBUS 校验和帧尾。
- **FreeRTOS 多任务架构**：串口接收、命令路由、机械臂动作、状态上报等任务分离，避免阻塞式串口读取。
- **安全机制**：支持急停命令、底盘 STOP 优先处理、心跳超时检测、状态上报和错误计数。

## 核心模块说明

### `ESP_ground`

地面站固件负责连接 TJC 串口屏和无线串口模块。屏幕按钮事件先被解析为本地命令码，再由 `CommandMapper` 映射为统一协议功能码，最后通过 LoRa / 无线 TTL 串口发送给车载主控。

地面站还会接收车载端返回的心跳、状态上报、ACK/NACK 和视觉识别结果，并维护车辆在线状态、最近识别结果、错误计数等信息。

### `ESP_planner`

车载主控是系统中心节点，负责：

- 解析来自地面站的控制指令；
- 接收 Jetson Nano 的视觉识别结果；
- 将底盘类命令转发给 STM32F103；
- 将机械臂类命令投递给机械臂任务执行；
- 通过众灵 ZP ASCII 协议控制总线舵机；
- 定时向地面站发送心跳和状态上报；
- 在急停时停止底盘、停止舵机并释放扭力。

### `Servo_test`

舵机测试工程用于验证众灵 ZP 系列总线舵机的串口接线、ID、波特率、位置读取、释力/恢复扭力和单舵机移动命令。当前工程已确认使用众灵 ASCII 字符串协议，例如：

```text
#000P1500T1000!
{#000P1500T1000!#001P1600T1000!}
#000PULK!
#000PULR!
```

## 通信协议概览

系统内部使用统一二进制串口协议，基本帧格式如下：

```text
AA 55 | Version | SrcAddr | DstAddr | MsgType | FuncCode | SeqNum | DataLen | Data | CRC16_L CRC16_H | 0D 0A
```

协议要点：

- 帧头：`0xAA 0x55`
- 帧尾：`0x0D 0x0A`
- 最大数据区：64 字节
- 校验：CRC-16/MODBUS，低字节在前
- 设备地址：
  - `0x20`：车载 ESP32-S3 主控
  - `0x30`：Jetson Nano
  - `0x40`：STM32F103 底盘
  - `0x50`：地面站
  - `0x60`：舵机控制板逻辑地址
- 功能码分区：
  - `0x00 ~ 0x0F`：系统命令、心跳、状态、急停
  - `0x10 ~ 0x1F`：底盘运动
  - `0x20 ~ 0x2F`：机械臂动作
  - `0x30 ~ 0x3F`：视觉识别信息

完整协议说明见：

- [`ESP_ground/docs/通信协议规范.md`](ESP_ground/docs/通信协议规范.md)
- [`ESP_planner/docs/通信协议规范.md`](ESP_planner/docs/通信协议规范.md)

## 硬件组成

| 模块 | 作用 |
|---|---|
| ESP32-S3 地面站 | 读取 TJC 串口屏按钮事件，发送控制命令，显示状态 |
| ESP32-S3 车载主控 | 多串口中心路由、机械臂控制、状态管理 |
| TJC 串口屏 | 人机交互界面 |
| 无线 TTL / LoRa 串口模块 | 地面站与车载主控之间的无线链路 |
| Jetson Nano + USB 摄像头 | 视觉识别结果输入 |
| STM32F103 + 麦克纳姆轮底盘 | 车辆运动执行 |
| 众灵 ZP 总线舵机控制板 | 双机械臂舵机驱动 |
| 众灵 ZP 总线舵机 | 双机械臂关节与夹爪 |

## 开发环境

- PlatformIO
- Arduino framework for ESP32
- FreeRTOS（Arduino-ESP32 内置）
- VS Code + PlatformIO IDE 扩展
- 串口监视器波特率：`115200`

三个 PlatformIO 工程当前均使用：

```ini
[env:freenove_esp32_s3_wroom]
platform = espressif32
board = freenove_esp32_s3_wroom
framework = arduino
```

## 调试与联调建议

1. 先分别烧录并验证 `Servo_test`、`ESP_ground`、`ESP_planner`，确认单模块工作正常。
2. 舵机必须使用外部供电，并与 ESP32、舵机控制板共地。
3. 所有 UART 连接优先检查 TX/RX 是否交叉、GND 是否共地、波特率是否一致。
4. 先只连接调试 USB，确认启动日志正常，再逐步接入无线串口、Jetson、STM32 和舵机控制板。
5. 地面站移动按钮遵循“按下发送方向、弹起发送 `CHASSIS_STOP`”的遥控逻辑。
6. 修改机械臂动作组前，先逐个舵机确认安全 PWM 范围，再写入 `ESP_planner/src/arm/ArmActionGroups.cpp`。
7. 急停和底盘 STOP 应作为联调重点，先验证停止链路，再测试连续运动。
